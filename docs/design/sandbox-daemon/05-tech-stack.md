Agent Python runs as RustPython compiled to WASM. |
| Framework / runtime | A Rust daemon exposing a **local control socket** (native RPC + a single-tool MCP front door) over UDS / named pipe; **Wasmtime** WebAssembly engine embedding **RustPython** compiled to `wasm32` (sandbox) | No network transport; a capability runtime with no ambient authority; the agent Python contract is the standard-library data-manipulation surface plus the broker SDK — no third-party or native packages. |

## Isolation boundary (single, cross-platform)

The sandbox boundary is the **WebAssembly capability model**, enforced by the Wasmtime runtime, not per-OS isolation primitives. This is the load-bearing security control (ADR-013) and it is identical on Linux, macOS, and Windows:

- **No ambient authority.** A WASM guest cannot issue a syscall; it can only call host functions (imports) the Rust runtime explicitly links. The runtime links exactly one — the broker call shim — and grants **no** WASI socket, host-filesystem, or process-spawn capability. Network egress is therefore *structurally absent*: there is no `socket()` to reach, so `ctypes`/raw-syscall escape (the dominant residual under the previous OS-isolation design) does not apply — WASM has no FFI path to native code.
- **No subprocess.** WASM has no process model, so the guest cannot spawn a child — SR-7's subprocess clause is satisfied structurally rather than by a Job Object or `process-fork` deny rule.
- **Resource limits in-runtime.** Wasmtime **fuel** bounds CPU, **maximum linear memory** bounds RAM, and an **epoch-based deadline** bounds wall-clock — replacing OS `rlimit`/Job-Object limits with cross-platform runtime limits (SR-8).
- **The OS still runs the Rust host**, but it is no longer the isolation boundary. Per the *WASM-as-sole-boundary* decision, Seatbelt / AppContainer / seccomp are **not** layered around the host; the runtime is the single boundary and Wasmtime is the trusted computing base. The residual — a Wasmtime sandbox escape — is tracked in [10 — Risks](./10-risks.md) (it replaces the macOS/Windows egress-lockdown risk and OQ-7).
